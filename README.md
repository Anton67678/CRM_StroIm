# CRM СтройИм — AI-Помощник

## Описание
CRM-система для управления ремонтно-строительными проектами с интегрированным AI-ассистентом на базе Qwen 2.5 (OpenRouter).

## Компоненты Архитектуры

### 1. AI-Агент (Qwen 2.5 + OpenRouter)

**Модель:** `qwen/qwen-2.5-72b-instruct` через OpenRouter API

**Возможности:**
- ✅ Ответы на вопросы по строительству и ремонту
- ✅ Формирование отчётов из базы данных CRM
- ✅ Генерация документов (сметы, акты, планы)
- ✅ Консультации по СНиП, ГОСТ, технологиям
- ✅ Рассылка задач мастерам в Telegram

**Протокол взаимодействия:**
```
Пользователь → Telegram/Web Chat → FastAPI → AI Agent → PostgreSQL/RAG → Ответ
```

---

### 2. Чат внутри CRM (Web Chat)

**Расположение:** Плавающий виджет поверх дашборда CRM

**Технологии:**
- Frontend: Vanilla JS + CSS (интеграция в `index.html`)
- Backend: Server-Sent Events (SSE) для потокового ответа
- API: `POST /api/chat/stream`

**Функционал:**
- Одновременный просмотр дашборда и общение с ботом
- Контекстные подсказки (брат данные из текущей вкладки)
- История диалогов (сохранение в БД)

---

### 3. Function Calling (Работа с данными CRM)

Бот умеет вызывать внутренние API CRM для получения данных:

| Инструмент | Описание | Пример запроса |
|------------|----------|----------------|
| `get_objects()` | Список всех объектов | "Покажи все активные объекты" |
| `get_financials(object_id)` | Финансы объекта | "Прибыль по Беговой" |
| `get_debtor_list()` | Долги клиентов/подрядчиков | "Кто должен денег?" |
| `get_acts(object_id)` | Акты по объекту | "Покажи акты для объекта 5" |
| `get_contractor_works()` | Работы подрядчиков | "Что делает Шухрат?" |

**Механизм:**
1. AI определяет намерение пользователя
2. Выбирает нужный инструмент
3. Вызывает внутренний API (`http://backend:8000/...`)
4. Анализирует JSON-ответ
5. Формирует понятный ответ

---

### 4. RAG (Retrieval Augmented Generation) — База Знаний

**Назначение:** Углублённые ответы по строительным нормам, технологиям, материалам

**Структура:**

```
knowledge/
├── snip/               # СНиПы, ГОСТы
│   ├── snip_31-01-2003.pdf
│   └── gost_...pdf
├── materials/          # Характеристики материалов
│   ├── gvlv_specs.md
│   └── drywall_types.md
├── processes/          # Технологические карты
│   ├── soundproofing.md
│   └── waterproofing.md
└── company/            # Внутренние регламенты
    └── standards.md
```

**Пайплайн:**

```
1. Загрузка документа → 2. Разбивка на чанки → 3. Векторизация (Embeddings) → 4. Сохранение в pgvector

5. Запрос пользователя → 6. Поиск похожих чанков → 7. Передача в Qwen + контекст → 8. Ответ
```

**Технологии:**
- Хранение: PostgreSQL + расширение `pgvector`
- Embeddings: `text-embedding-3-small` (OpenAI) или локальная модель
- Поиск: Cosine similarity по векторам

---

### 5. Генерация Документов (JSON → DOCX/PDF)

**Сценарий:**
1. Бот собирает данные через диалог (размеры, материалы, цены)
2. Формирует JSON-структуру
3. Бэкенд подставляет данные в шаблон
4. Возвращает готовый файл

**Шаблоны:**
- `templates/smeta.docx` — Смета работ
- `templates/act.docx` — Акт выполненных работ
- `templates/estimate_shkaf.docx` — Расчёт шкафа/гардеробной
- `templates/report_month.pdf` — Месячный отчёт

**Библиотеки:**
- `docxtpl` — генерация Word-документов
- `weasyprint` — генерация PDF из HTML
- `openpyxl` — экспорт в Excel (уже есть в CRM)

---

### 6. Telegram Bot — Рассылки и Уведомления

**Функции:**
- ✅ Рассылка задач мастерам
- ✅ Уведомления о дедлайнах
- ✅ Отчёты по объектам
- ✅ Обратная связь от мастеров (фото, статус)

**Лимиты Telegram:**
- 30 сообщений/секунду (технический)
- Только пользователям, нажавшим `/start`
- Риск бана за спам (нужна кнопка "Отписаться")

**Планировщик:**
- `APScheduler` — cron-задачи для авто-рассылок
- Пример: *"Каждое утро в 08:00 — напоминание о задачах на сегодня"*
